- **Fix:** Normalize to "Log in" (matches canonical nav).

### H6. Radio + checkbox alignment broken on `/partner-apply`
- **Cause:** `_shared/_partials.css:145` applies `form.simple-form input { width:100%; padding:12px 14px }` to radios and checkboxes. The page's local override `.check-opt input, .radio-opt input { width:16px }` loses on CSS specificity (`form.simple-form input` = 0,1,2 beats `.radio-opt input` = 0,1,1).
- **Visual:** Radios/checkboxes stretch full width with 12px padding, label and input end up on opposite sides of the row.
- **Fix:** Either (a) scope the partials rule to text-type inputs only: `form.simple-form input:not([type="radio"]):not([type="checkbox"])`, or (b) raise override specificity. (a) is cleaner.

### H7. `partner-apply` function ignores half the form data
- **Form sends:** `full_name, studio_name, email, phone, city, state, zip, full_address, products[], average_project_size, clients_per_year, extra_profile_notes, contact_method, consent_text`
- **Function writes to structured columns:** name, email, phone, company, address (composed), avg_project_size, clients_per_year on `leads`; products → `partners.specializes_in`, avg → `partners.avg_job_size`
- **NOT written structurally:** `city`, `state`, `zip` (only inside composed address); `contact_method`; `extra_profile_notes` (despite the column existing on `partners`)
- **Effect:** Anything not structurally stored is recoverable only by digging through `lead_events.payload` jsonb — admin can't filter by it.
- **Fix:** Update `netlify/functions/partner-apply.js` to write `extra_profile_notes` to `partners.extra_profile_notes` (column exists per migration 20260518); decide whether to add structured columns for city/state/zip and contact_method, or accept jsonb-only storage.

### H8. Promo A public wording NOT reconciled (6 files, 7 occurrences)
- **PRD decision:** Client pays full price; designer's commission boosted to 40% (35% + 5%) on first deal, capped at $5,000 coverage value.
- **Current wording (client-side framing — wrong):**
  - `index.html:1339` "Approved studios receive their first plan complimentary…"
  - `index.html:1605` (popup) "we'll cover your first client's plan, complimentary"
  - `partner/index.html:186` "first plan complimentary — up to $5,000 of coverage"
  - `partnership/index.html:39` "receive their first plan complimentary"
  - `plans/index.html:1303` "the first plan is complimentary"
  - `plans/index.html:1600` "your first plan is complimentary"
  - `dashboard/index.html:867` "receive their first plan complimentary"
- **Fix:** Rewrite each instance to the designer-side framing. Suggested copy: *"Your first specified deal earns a boosted 40% commission (the value of a free plan, up to $5,000 of coverage). Claim it on your dashboard."*
